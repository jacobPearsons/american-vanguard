# AI System Bootstrap
### One-Prompt Production System Generator

This document defines how AI must generate a **complete production-ready project from zero**.

The system bootstrap allows developers to describe an application at a high level and AI will generate:

- full repository structure
- frontend application
- backend API
- authentication
- database schemas
- security configuration
- CI/CD pipelines
- observability
- deployment configuration

All generated systems must follow the architecture rules defined across the documentation system.

---

# Core Objective

The AI System Bootstrap converts a **single structured prompt** into a complete engineering system.

The generated project must be:

production-ready  
secure by default  
observability enabled  
cleanly structured  
scalable

The project must integrate with all internal architecture documents.

# Frontend System Generation

The frontend must include:

application layout  
component system  
API integration layer  
global state management  
animation integration

Requirements:

Tailwind CSS styling  
Framer Motion animation layer  
modular components

Frontend must follow frontend-lifecycle.md and component-design-rules.md.

---

# Backend System Generation

Backend must follow backend-request-lifecycle.md and backend-architecture.md.

Backend generation must include:

Express server setup  
routing layer  
controllers  
service layer  
data models  
validation middleware

Controllers must remain thin.

Business logic must exist only in services.

---

# Authentication System

Authentication must be generated by default.

Requirements:

session management  
secure cookies  
login and logout endpoints  
protected routes

Authentication must integrate with Passport strategies.

Password hashing must use a secure algorithm.

---

# Database Generation

AI must generate initial database schemas.

Database tasks include:

user model  
role model  
audit logs  
indexes

Database must support migration workflows.

Relational data should use PostgreSQL.

Flexible event data may use MongoDB.

---

# Security Configuration

Security must be enabled by default.

Required protections:

input validation  
rate limiting  
secure session cookies  
CSRF protection  
authorization middleware

Security rules must follow security-hardening.md.

---

# Observability System

The system must include observability infrastructure.

Required capabilities:

structured logging  
error reporting  
performance tracing

All backend requests must emit logs.

Observability must follow observability-architecture.md.

---

# CI/CD Pipeline

The generated system must include CI/CD automation.

Pipeline stages:

install dependencies  
run linting  
run tests  
build project  
deploy

CI/CD configuration must be placed in infrastructure/ci.

---

# Container Architecture

The system must support containerized deployment.

Docker configuration must include:

frontend container  
backend container  
database container

Containers must communicate through a defined network.

---

# Environment Configuration

AI must generate environment configuration templates.

Example variables:

DATABASE_URL  
SESSION_SECRET  
API_BASE_URL

Environment files must never contain secrets.

Secrets must be injected through deployment environments.

---

# Testing Infrastructure

The system must include automated testing.

Testing framework:

Jest

API tests  
service tests  
validation tests

Testing infrastructure must run automatically in CI.
